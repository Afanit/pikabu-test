# Функциональное описание (основные моменты):

Приложение состоит из двух вкладок: лента из всех постов и лента избранного. Есть возможность просмотра поста на отдельном экране.
## Основная лента:
- загрузка данных происходит целиком, без пагинация: поскольку данные из API даются одним куском, она бессмысленна
- посты показываются по свежести (по убыванию id)
- если постов нет, показывается соответствующая надпись
- при скролле вверх снизу появляется навигационный таб, скрывающийся при обратном скролле
- добавление в избранное по клику не меняет состав ленты
## Лента избранного:
- посты отсортированы по времени добавления в избранное
- если избранных постов нет, показывается соответствующая надпись
- избранные посты отмечены оранжевым
- при удалении из избранного пост удаляется из ленты
## Превью поста:
- переход по клику в заголовок открывает пост в отдельном окне с возможностью проскролла контента
- снизу появляется плашка с кнопками «назад» и «избранное» не убирается при скролле
- при открытии поста идёт запрос в API, на случай если данные изменились
## Отрисовка картинок:
- поскольку в API отсутствуют размеры картинки, размер плейсхолдера может «скакать» относительно реального размера изображения

# Техническое описание:
- Язык: Kotlin 1.4
- Система сборки: Gradle
- Минимальный API: 21+
- Архитектура: SingleActivity + MVVM (ViewModel + LiveData)
- DI: Dagger2 (чистый, не DaggerAndroid)
- Работа с сетью: Retrofit 2
- База данных: Room (вместе с LiveData)
- UI: ViewPager2 с RecyclerView (поправил текущий баг с диагональным проскроллом при одновременном наличии горизонтального+вертикального направлений) + TabLayout, обновление через SwipeRefreshLayout, в качестве корня использовал ConstraintLayout.
- View Binding: использовал официальную реализацию, а не Kotlin Synthetic, поскольку она гарантирует решение проблемы с отсутствием нужной вью в рантайме
- Картинки: Glide4 (у библиотеки есть баг с отображением изображения размером с плейсхолдер, официальный фикс с dontAnimatie() не работает)
- Анимация: CoordinatorLayout (нижний таб), TransactionManager для фрагментов и ValueAnimator для анимации элементов
- Rx: не потребовалось, LiveData было вполне достаточно
- VCS: Git, поскольку всё делалось за один подход и коммиты не были «чистыми», слил все изменения в один кормит
- Unit/UI Tests: отсутствуют. Поскольку большая часть функционала касается интерфейса, приложение отсматривалось силами асессоров (=руками) на 5-ом и 9-ом андроидах

# Будущие правки:
- Убрать самописную сериализация в строку, заменив на one-to-many через @Relation.
- Выделить отдельный слой данных, сейчас данные из сервера запрашиваются из ViewModel вместо RemoteRepository
- Использовать стандартный ListAdapter
